API Документация

Аутентификация

API поддерживает два способа аутентификации:

  • Bearer токен — передайте JWT в заголовке Authorization: Bearer <token>
  • API ключ (только Pro) — передайте ключ в заголовке X-API-Key: <key>

🔑 Аутентификация

POST /api/auth/register Регистрация нового пользователя
{"email": "user@example.com", "password": "min8chars", "full_name": "Имя"}
POST /api/auth/login Получить JWT токен
{"email": "user@example.com", "password": "password"}
GET /api/auth/me Информация о текущем пользователе
POST /api/auth/regenerate-api-key Сгенерировать новый API ключ

🎨 Шаблоны

GET /api/templates/ Список шаблонов

Параметры: marketplace, offset, limit

POST /api/templates/ Создать шаблон
{"name": "Мой шаблон", "marketplace": "wb", "canvas_json": "{...}", "variables": "[...]", "canvas_width": 900, "canvas_height": 1200}
GET /api/templates/{uid} Получить шаблон по UID
PUT /api/templates/{uid} Обновить шаблон
DELETE /api/templates/{uid} Удалить шаблон
POST /api/templates/{uid}/preview Сгенерировать превью шаблона

⚡ Генерации

POST /api/generations/ Создать генерацию изображения
{"template_uid": "uuid", "input_data": {"title": "Текст"}, "output_format": "png", "output_width": null, "output_height": null}
GET /api/generations/ Список генераций
GET /api/generations/{uid} Статус генерации
GET /api/generations/{uid}/download Скачать результат
DELETE /api/generations/{uid} Удалить генерацию

Пример использования

curl -X POST https://api.kartochka.ru/api/generations/ \ -H "X-API-Key: ваш-api-ключ" \ -H "Content-Type: application/json" \ -d '{ "template_uid": "uuid-шаблона", "input_data": { "title": "Кроссовки Nike", "price": "4990" }, "output_format": "png" }'
